## 一、 表格标签 (`<table>` 及其家族)

在早期的 Web 时代，开发者经常用表格来进行网页排版（比如把导航栏放在左边的格子里，内容放在右边的格子里），这是一种非常糟糕的做法。如今，表格已经回归了它的本职工作：展示规整的二维数据（就像 Excel 那样）。

构建一个语义化良好的表格，需要多个标签配合：
- `<table>`：表格的外层容器。
- `<thead>`：表头区域（通常包含列名）。
- `<tbody>`：表格的主体数据区域。
- `<tr>` (Table Row)：定义表格中的一行。
- `<th>` (Table Heading)：定义表头单元格（文字通常默认加粗并居中）。
- `<td>` (Table Data)：定义普通的数据单元格。

**示例结构:**
```html
<table>
  <thead>
    <tr>
      <th>SKU 编号</th>
      <th>货物名称</th>
      <th>库存状态</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>WH-A-001</td>
      <td>人体工学椅</td>
      <td>库存充足</td>
    </tr>
    <tr>
      <td>WH-B-042</td>
      <td>机械键盘</td>
      <td>需补货</td>
    </tr>
  </tbody>
</table>
```

## 二、 表单标签 (`<form>` 及其控件)

表单是网页与用户进行数据交互的核心桥梁。无论是登录注册、发博客评论，还是提交问卷，都离不开表单。
- `<form>`：表单的容器。它通常带有 `action`（数据提交到哪个服务器地址）和 `method`（提交方式，如 GET 或 POST）属性。
- `<input>`：最百变的核心控件（单标签）。通过改变 `type` 属性，它可以变成各种样子：
    - `type="text"`：单行文本框。
    - `type="password"`：密码框（输入内容会变成小黑点）。
    - `type="radio"`：单选按钮。
    - `type="checkbox"`：复选框。
    - `type="file"`：文件上传按钮。
- `<label>`：为表单控件定义说明文字。将它与 `<input>` 绑定后，点击文字就能自动聚焦到对应的输入框，极大提升用户体验。
- `<textarea>`：多行文本输入区（适合写文章或长评论）。
- `<button>`：按钮。如果放在 `<form>` 里面，默认具有提交表单的功能。

## 三、 原生交互标签 (`<details>` 与 `<summary>`)

以前我们要实现“点击展开/折叠”的手风琴效果，必须写 JavaScript 代码。但在现代 HTML5 中，你可以直接使用这两个标签实现原生交互。
- `<details>`：创建一个可展开隐藏内容的挂件。
- `<summary>`：定义挂件可见的标题。点击它即可展开或收起剩余内容。

**示例结构:**
```html
<details>
  <summary>点击查看雅思备考计划</summary>
  <p>1. 每天背诵 100 个核心词汇。</p>
  <p>2. 每周完成两套阅读真题。</p>
</details>
```
